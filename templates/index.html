Attack Methods

Four adversarial attacks — two gradient-based (ML) and two evolutionary (EC)

⚡

FGSM

Fast Gradient Sign Method — Single-step attack that perturbs inputs in the direction of the loss gradient. Fast but less powerful.

ML · Gradient-Based
🔄

PGD

Projected Gradient Descent — Iterative multi-step attack within an ε-ball. The gold standard for evaluating adversarial robustness.

ML · Gradient-Based
🧬

Genetic Algorithm

Population-based evolutionary attack using selection, crossover, and mutation. Gradient-free — bypasses gradient masking defenses.

EC · Gradient-Free
🔬

Differential Evolution

Continuous optimization in reduced-dimension space. Uses mutation/recombination to find adversarial perturbations without gradients.

EC · Gradient-Free

Defense Strategies

Four defense mechanisms — proactive and reactive approaches compared

🛡️

Adversarial Training

Retrains the model with PGD-generated adversarial examples. Most effective known defense but computationally expensive.

Proactive · Retraining
🔄

Input Transformation

Pre-processes inputs with smoothing, bit-depth reduction, and compression to remove adversarial perturbations before classification.

Reactive · No Retraining
🔍

Detection Network

Binary classifier trained on CNN features to distinguish clean from adversarial inputs. Rejects detected attacks before classification.

Reactive · Detection
🧪

Defensive Distillation

Teacher-student training with temperature scaling. Smooths decision boundaries to reduce gradient magnitudes exploited by attackers.

Proactive · Distillation

How It Works

The evaluation pipeline

1️⃣

Train Models

Train a CNN on Indian Traffic Signs (58 classes), then apply each defense strategy to create 4 defended variants.

2️⃣

Generate Attacks

Run all 4 attack methods against each defense to create a comprehensive attack-defense matrix.

3️⃣

Analyze Results

Compare success rates, robust accuracy, perturbation metrics, and defense effectiveness visually.
